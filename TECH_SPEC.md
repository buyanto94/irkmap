# Типы пользователей

**Администратор**- Полный доступ, управление пользователями (бан/удаление), создание категорий, назначение модераторов.

**Модератор** - Создание категорий, верификация заявок на регистрацию бизнесов, проверка и публикация/отклонение объектов.

**Бизнес-пользователь** - Регистрация (через подтверждение), создание/редактирование своих объектов в ЛК, управление профилем компании.

**Пользователь**  - Регистрация (через подтверждение), управление личным профилем, просмотр и удаление избранных объектов.  

# Регистрации бизнес пользователей

Процесс регистрации разделен на два этапа для защиты от спама:

1. Заявка: Пользователь заполняет форму
   - Email
   - Название компании
   - ИНН/Реквизиты
   - Контактное лицо
   - предусмотреть возможность загрузки документов в произвольной форме, не больше 5 документов).

   В этот момент отправляется письмо на почту, где пользователь подтверждает заявку на регистрацию. Email переходит в статус Confirmed. Статус аккаунта: Pending.

2. Проверка: Модератор получает уведомление, проверяет данные и нажимает «Подтвердить». Статус аккаунта: Active.
	
   В случает если у модератора возникают вопросы он переводит статус в AdditionalValidation
  
# Функционал личного кабинета бизнес-пользователя

Бизнес-пользователь видит список своих объектов с явным указанием статуса.
Кнопка создания объекта.
При создании объекта  Бизнес-пользователь вводит следующие данные
- Категория: Выбор из выпадающего списка (создается заранее модератором).
- Название: Текстовое поле (до 100 символов).
- Описание: Rich Text Editor (жирный текст, списки).
- Фотогалерея: Загрузка нескольких изображений (drag-and-drop) с возможностью выбора главного фото.
- Геолокация: * Интерактивная карта (Open Street Map).
  - Пользователь ставит маркер, координаты сохраняются в БД автоматически.
 
После сохранения объекта  его статус меняется на «На модерации»
 
# Функционал личного кабинета Модератора
 
Каждый модератор закреплен за определенным регионом.
 
## Модерации контента                                             
 
В данном разделе отображаются все объекты в статусе на Модерации.
Модератор может подтвердить изменения объекта (статус: Готов к публикации), отправить на корректировку (статус: Требует корректировки), в этом случае пользователь вносит изменения и отправляет объект на повторную модерацию, или отклонить объект (статус: Отклонено)
- Интерфейс модератора: Список объектов, ожидающих проверки.
- Действия: «Опубликовать» «Отклонить» «Отправить на корректировку» (для двух последних действий нужно обязательно указать причину в текстовом поле, которая уходит письмом владельцу).

Вверху списка есть система фильтрации – модератор выбирает нужный статус объекта для вывода

Инструментарий модерации:
- Режим сравнения (Diff View): Если объект уже был опубликован, но пользователь внес изменения, модератор должен видеть «Старую» и «Новую» версии (выделение измененных полей/фото).
- Управление статусами:
  - Готов к публикации: После нажатия объект переходит в финальный статус. У бизнес-пользователя в ЛК появляется возможность нажать кнопку «Опубликовать» (чтобы он мог сам выбрать время запуска).
  - На корректировку: Обязательное поле «Комментарий модератора» (например: «Замените главное фото, оно слишком мелкое»). Статус объекта меняется на Требует корректировки, пользователю открывается доступ на редактирование.
  - Отклонить: Полный отказ в публикации (например, если объект не соответствует тематике портала).
 
## Верификация бизнес-пользователей

Экран для обработки новых заявок на регистрацию.
- Очередь регистрации: Список пользователей со статусом Не активен, у которых Email уже перешел в статус Подтвержен.
- Карточка проверки: Просмотр данных (ИНН, реквизиты, контактное лицо) и блока вложенных документов (до 5 файлов).
- Инструменты модератора:
  - Кнопка «Подтвердить»: Переводит аккаунт в Active, пользователю отправляется Welcome-письмо.
  - Кнопка «Отклонить»: Вызывает модальное окно для ввода причины. Аккаунт удаляется или архивируется, пользователю уходит отказ.
 
# Функционал личного кабинета Администратора
 
## Управление категориями объектов
 
- Создание/Редактирование:
  - Наименование: Текстовое поле.
  - Иконка: Загрузка файла (SVG/PNG) 
  - URL Slug: Поле с автогенерацией из названия (транслит), доступное для ручной правки. Валидация на уникальность.
- Сортировка: Возможность менять порядок категорий для отображения в сетке на главной странице (Drag-and-drop или поле «Вес»).
 
## Контроль и Журналирование

Все изменения объектов храниться в базе данных и выводятся в данном разделе.
 
Главная страница админа:
Выводится следующая статистическая информация:
- сколько объектов на модерации
- сколько активных бизнесов,
- самые популярные категории.
 
# Публичная часть
 
## Главная страница
- Строка быстрого поиска: Поиск по названию или ключевым словам.
- Сетка категорий: Иконки (Гостиницы, Активный отдых, Культура и т.д.), заданные модератором.
- Блок «Популярное» или «Новое»: Объекты с самым высоким рейтингом или недавно добавленные.
- Все страницы публичной части должны поддерживать разметку Open Graph

## Поиск

При поиске информации будет использоваться технология Sphinx с поддержкой морфологического анализа для корректной обработки русского и английского языков (поиск по корню слова, учет падежей и чисел), нечеткого поиска (Fuzzy Search) для нахождение результатов при наличии 1-2 опечаток в слове и ранжирования для выдачи наиболее релевантных совпадений в начале списка.
 
## Страница каталога (Список + Фильтры)
Интерфейс, где пользователь проводит основное время. сплит-экран: слева фильтры в центре список карточек, снизу — интерактивная карта.
Система фильтрации:
- По категориям: Множественный выбор (через чекбоксы).
- По удаленности: (Если есть доступ к геолокации пользователя) — «В радиусе 10/50/100 км».
- По рейтингу: Сортировка от высокого к низкому.
- Текстовый поиск: Живой поиск (по мере ввода букв).

Карточка объекта (в списке):
- Превью фото.
- Название и Категория.
- Короткое описание (2-3 строки).
- Рейтинг (звезды).
- Кнопка «Показать на карте» (центрирует карту на объекте).

## Интерактивная карта (Map View)
Ключевой элемент для туристического портала.
- Кластеризация: Если в одной точке (например, в городе) 50 отелей, они должны группироваться в кружок с цифрой «50», который распадается при приближении.
- Балуны (Pop-ups): При клике на маркер открывается мини-карточка с фото, ценой (если есть) и ссылкой на полную страницу.
- Поиск в области видимости: Кнопка «Искать в этом районе» (обновляет список объектов при перемещении карты).
 
## Детальная страница объекта
Максимум информации, которую заполнил бизнес-пользователь.
1. Галерея: Полноэкранный просмотр фотографий.
2. Инфо-блок: Название, категория, статус (работает/закрыто).
3. Описание: Полный текст с форматированием.
4. Контакты: Кнопка «Показать телефон» или «Перейти на сайт владельца» (для аналитики кликов).
5. Карта проезда: Статичная карта с точкой и возможностью построить маршрут.
 
# Технические требования
- Стек: PHP (Laravel, Symfony)
- База данных: MariaDb
- Карты: Open Street Map, в крайнем случае Яндекс
- ЧПУ (Человекопонятные URL): Вместо site.com/object/123 использовать site.com/hotels/gostinica-uyut.
- Lazy Loading: Изображения в каталоге подгружаются только при скролле (чтобы сайт не «тормозил»).
- Адаптивность: Мобильная версия обязательна, так как 80% туристов ищут объекты «на ходу» со смартфонов.
